K Ztelnet Patch v0.0.1

0. 소개

  이 프로그램은 현재 공개되어 있는 Ztelnet for OS/2 public beta 2 가 가지고
있는 몇가지 벌레를 제거하고, 제한적이나마 세벌식 자판을 쓸 수 있도록 해 주는 
프로그램입니다.  

1. 제작 환경 

  - 한국어판 OS/2 Warp 4 with FixPak #5

  - EMX 0.9d fix #4 + gcc 2.8.1
  
2. 실행에 필요한 환경

  제작 환경을 만족시킨다면 상관이 없습니다만, 특정 OS/2 에 맞추어진 코드가
없기 때문에, OS/2 가 동작하는 곳이라면 어느 곳에서나 실행되리라고 봅니다.
  
3. 기능

  - 사용하는 키보드 자판에 상관없이 한글 입력이 가능합니다.
  
  - 세벌식 390 자판을 지원합니다.

  - 한글 입력 상태에서 채팅 라인을 쓸 때 사이띄우개를 두번 눌러야 하는 벌레 
    패치합니다

  - Ztelnet/2 가 입력 상태일 때에만 상태 상자가 나타납니다
  
4. 설치

  압축 파일을 아무 디렉토리에다 풀면 됩니다. ^^ 단, kzphook.dll 은 LIBPATH 
가 지정되어 있는 디렉토리에 넣어두면 됩니다.
  
5. 사용법
  
  WPS 나 명령행에서 kzp.exe 를 실행시키면 됩니다.
  
5-1. 사용되는 단축키

  Ctrl-Space : KZP 의 한/영 상태를 변환합니다.
  Ctrl-F3    : KZP 의 글자/라인 상태를 변환합니다.
  
5-2. 사용되는 마우스 기능

  오른쪽 버튼 : 팝업 메뉴가 나타납니다.
  왼  쪽 버튼 : 한/영 또는 글자/라인 상태를 변환시킵니다.
  
5-3. 메뉴

   Hide    : KZP 상태 상자를 강제로 숨깁니다.
   Options : KZP 가 패치할 기능을 선택합니다.
   Exit    : KZP 의 실행을 끝냅니다.
   
5-4. 명령행 옵션

   "--no-3bul"    : 세벌식 패치를 하지 않습니다.
   "--no-chatline : 채팅 라인 패치를 하지 않습니다.
     
6. 제한 사항 또는 알려진 벌레

  - 어느 윈도의 클래스 이름이 "ClientWindowClass" 일 경우, KZP 가 오동작을 할 
    수 있습니다.
    Ztelnet 의 클라이언트 윈도의 클래스 이름이 "ClientWindowClass" 이고, KZP
    에서 현재 윈도가 Ztelnet 인지 판단하기 위해 윈도의 클래스 이름을 쓰기 때문
    입니다.
    
  - 하나의 Ztelnet 세션만을 지원합니다.
    예를 들어, 여러 개의 Ztelnet 세션이 한 꺼번에 실행되었을 경우에, 각각에
    대한 한/영 및 글자/라인 상태를 기억하지 않습니다. 하나의 Ztelnet 세션이 
    한글 입력 상태였을 때, 다른 하나가 영문 입력 상태였다면, KZP 는 그 상황
    을 자동으로 판단하지 않습니다. 따라서 KZP 의 상태를 강제로 변환시켜야 
    합니다.

  - Ztelnet 세션과 KZP 사이에 한/영 및 글자/라인 상태 동기화가 이루어지지 
    않을 수 있습니다.
    이것은 Ztelnet 이 접속되어 있지 않은 경우, 키입력을 처리하지 않지만, 
    KZP 에서는 접속 여부를 확인할 수 없기 때문에 일어나는 문제입니다. 따라서,
    KZP 의 상태를 강제로 변환시켜야 합니다.

  - 세벌식으로 입력할 때 도깨비불 현상이 나타납니다.
    기본적으로 두벌식 입력 방식에 키맵핑 형식으로 세벌식 입력을 구현하였기 
    때문에 나타나는 현상입니다.
      
  - Task List(창목록)에서 KZP 를 활성화 시킬 경우, Task List 윈도가 사라지지 
    않습니다.
    이것은 KZP 상태 상자를 Frame Window 를 쓰지 않고, 일반 윈도를 쓴 탓에 태
    스크 스위칭에 관계된 메세지를 제대로 처리하지 못해서 생겨나는 현상인 것 
    같습니다.
